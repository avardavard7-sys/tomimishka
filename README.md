# ТОМИ МИШКА · French IT

Фото места + чертежи + текстовое ТЗ на входе → на выходе:

1. **Точный 3D-эскиз** — параметрическая модель (Three.js), собранная кодом из спецификации, которую извлекает Claude из чертежей и ТЗ. Размеры математически точные, крутится мышью, размерные линии как на чертеже, экспорт в многостраничный **PDF-эскиз** и **GLB**. Два режима:
   * **Предмет** — стойки, ресепшены, витрины, шкафы, стеновые панели.
   * **Интерьер** — комната целиком: пол, потолок, стены по сторонам света, карниз с LED, плинтус и обставленная мебель (кровати, диваны, ТВ, ковры, картины, светильники, растения). Стены строятся нормалью внутрь, поэтому снаружи виден «кукольный домик», а кнопкой **Внутри** камера заходит в комнату на уровень глаз.
2. **Фотореалистичный рендер** — Gemini 2.5 Flash Image редактирует исходное фото: меняет дизайн изделия, сохраняя ракурс, помещение и свет. Слайдер «до/после», кнопка правок и перегенерации.
3. **Фотореализм с любого ракурса** — крутишь 3D до нужного вида и жмёшь «✦ Фотореализм с этого ракурса»: снимок модели уходит в Gemini с жёстким требованием сохранить камеру и геометрию, поднимая только материалы, свет и детализацию. Кнопкой «← вернуться в 3D» возвращаешься к модели и крутишь дальше.
4. **Ретушь** (`/retouch`) — загрузил фото / ИИ-картинку / PDF / ZIP, написал что убрать («убрать стул слева и провода») — объект стирается, фон достраивается, остальной кадр остаётся пиксель в пиксель. Убирать можно по шагам, с откатом и слайдером «до/после».

Сам 3D-вьювер работает с отражениями окружения (RoomEnvironment PMREM), физически корректным полированным камнем (clearcoat), ambient occlusion, bloom на подсветке и ACES-тонмаппингом — чтобы модель выглядела рендером, а не серыми кубиками.

Форматы исходников: JPG / PNG / WEBP, PDF (постранично конвертируется в картинки), ZIP (вытаскиваются картинки и PDF). DWG в v1 не поддерживается — экспортируй чертёж в PDF.

## Запуск локально

```bash
cd tomi-mishka
npm install --legacy-peer-deps
cp .env.example .env   # и заполни ключи
npm run dev
```

## Переменные окружения

| Переменная | Обязательно | Что это |
|---|---|---|
| `ANTHROPIC_API_KEY` | да | мозг: анализ фото/чертежей → спек |
| `ANTHROPIC_MODEL` | нет | по умолчанию `claude-sonnet-4-6` |
| `GEMINI_API_KEY` | да (для фоторендера) | Google AI Studio → API key |
| `GEMINI_IMAGE_MODEL` | нет | по умолчанию `gemini-2.5-flash-image` |
| `IMAGE_PROVIDER` | нет | `gemini` (дефолт) или `fal` |
| `FAL_KEY` | нет | если `IMAGE_PROVIDER=fal` (Flux Kontext) |
| `NEXT_PUBLIC_SUPABASE_URL` | нет | без него всё работает, но проекты не сохраняются |
| `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` | нет | публикуемый ключ `sb_publishable_...` (поддерживается и `NEXT_PUBLIC_SUPABASE_ANON_KEY`) |

## Supabase

1. Создай проект, возьми URL и legacy anon key (`eyJ...`).
2. SQL Editor → выполни `supabase/schema.sql` (идемпотентный: таблица `design_projects`, RLS, бакет `design-projects` с политиками).
3. Auth → Providers → Email: рекомендуется отключить «Confirm email», чтобы вход был мгновенным.

## Деплой на Vercel

1. Запушь репозиторий, импортируй в Vercel.
2. Добавь все переменные окружения из таблицы выше.
3. Deploy. API-роуты имеют `maxDuration = 60` — укладывается в лимиты Hobby.

## Архитектура (почему «без обмана»)

- **Claude (vision)** читает чертежи и ТЗ и выдаёт строгий JSON-спек: участки фасада (`runs`), высоты 900/1200, толщины столешниц, выступы, цоколь, LED, материалы, модули, колонны, оборудование. Красные размеры с чертежа попадают в спек как есть.
- **Детерминированный движок** (`src/lib/scene.ts`) собирает геометрию из спека чистой математикой. ИИ не рисует эскиз «на глаз» — он заполняет параметры, геометрию строит код. Число в спеке = число в модели = число на размерной линии.
- **Нормализация** (`normalizeSpec`) гарантирует, что кривой ответ ИИ никогда не уронит вьювер.
- **Фоторендер** — генеративный (Gemini/Flux): для него предусмотрены правки и перегенерация до нужного результата.
